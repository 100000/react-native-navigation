A container view that manages a radio-style selection interface, where the selection determines which child view controller to display.

```js
{
  id: 'BOTTOM_TABS_LAYOUT',
  children: [
    {
      component: {
        id: 'HOME_SCREEN'
        name: 'HomeScreen'
      }
    },
    stack: {
      id: 'PROFILE_TAB',
      children: [
        {
          component: {
            id: 'PROFILE_SCREEN',
            name: 'ProfileScreen'
          }
        }
      ]
    }
  ]
}
```

## `id`

| Type   | Required | Description                                                                                                                                                    |
| ------ | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| string | No       | Unique id used when interacting with the view with the Navigation api, usually `Navigation.mergeOptions` which accepts the componentId as it's first argument. |

## `children`

| Type               | Required | Description                   |
| ------------------ | -------- | ----------------------------- |
| [Layout[]](layout) | YES      | Children layouts of any kind. |

## `options`

| Type                    | Required | Description                                                   |
| ----------------------- | -------- | ------------------------------------------------------------- |
| [Options](options-root) | No       | dynamic options which will apply to all screens in bottomTabs |
